# 整屏切换改造计划

> 目标:把"长滚动 + 滚动进度驱动动画"改成 **fullpage 整屏切换 + 进入即播放**,还原原站交互。

## 用户提的三点问题

1. **页面切换是整屏翻页,不是进度滚动**:鼠标滚一下 → 直接进入下一个完整页面(fullpage 式),而非滚动条按比例滚动。
2. **方法页 4 条卡动画应"依进入"触发**:翻到该屏时 4 张卡播放入场,**不依赖滚动进度**(去掉 pin/scrub)。
3. **作品不要自动轮播**:精选作品手风琴只在**鼠标 hover** 时展开,去掉 `setInterval` 自动切换。

## 根因

三点指向同一结论:原站是**整屏切换站**,不是长滚动站。当前所有"scrub / ScrollTrigger 进度驱动"的动画模型不匹配,需改为"翻到该屏 → 播放一次"。

## 改造方案(选型:Transform Fullpage)

用 `translateY` 移动一个 `.fp-track` 在各 100vh 段落间切换,而非真实文档滚动。

> 取舍:这样最贴合"整屏"心智,且超高段落可独立内部滚动;代价是 ScrollTrigger/v-reveal/v-countup 这些"滚动驱动"的逻辑要改成"active 进入驱动"——而这正是用户要的"依进入"。

### A. 新增 `composables/useFullpage.ts`
- 状态:`active`(当前屏索引)、`count`、`locked`。
- 监听 `wheel` / 触摸 / 方向键 → `goTo(active ± 1)`。
- `goTo`:GSAP 动 `.fp-track` 的 `y`(px),`power3.inOut`,切换中 `locked` 锁定 + 短冷却,避免触控板连发。
- **超高段落内部滚动**:当前屏 `scrollHeight > clientHeight` 且未到边缘时,放行原生滚动(不 `preventDefault`),到边缘再翻屏。
- resize 时重新对位 `y = -active * innerHeight`。
- 模块级单例导出 `goToIndex` / `goToId(id)`(映射段落 id→索引),供导航和按钮调用。

### B. 改 `App.vue`
- body/根 `overflow: hidden`;结构 `.fp-viewport`(fixed 100vh overflow hidden)>`.fp-track`(纵向排列)>6 段落。
- 给每个段落传 `:active="active === i"`。
- `SiteNav`、Contact 的"联系我/返回顶部"改用 `goToId`,替换 `scrollToId`。
- `ScrollProgress` 改为按 `active/count` 显示,或换成右侧屏点(dots)。

### C. 段落动画改为"active 进入驱动"
加一个小工具 `useEnter(activeRef, fn)`:active 第一次变 true 时执行一次入场。

| 段落 | 改法 |
|---|---|
| Hero | 保留入场 stagger(index 0 初始即播);Ken Burns 改成**持续缓动循环**,不再绑滚动;去掉滚动退场。 |
| Resume | 进入时:count-up 数字 + 各块 stagger 浮入(把 v-countup/v-reveal 改成 enter 驱动)。 |
| About | **去掉 pin/scrub/聚光**;进入时 4 张卡 stagger 浮入(对应问题 2)。 |
| Featured | **去掉 `setInterval` 自动轮播**,只保留 hover 展开(对应问题 3);默认折叠或默认展开第 1 张。 |
| Catalog | 进入时网格卡 stagger 浮入(v-reveal → enter 驱动)。 |
| Contact | 进入时玻璃卡/手机框淡入。 |

### D. 善后
- `useSmoothScroll`(Lenis)停用或移除;`scrollToId/scrollToTop` 由 `useFullpage` 的 `goToId` 取代。
- `useReveal` / `useCountUp`:从 ScrollTrigger 改为可被 `active` 触发(或提供命令式 API)。
- `SectionFlash`:scrub 失效,改为翻屏时触发一次闪屏,或暂时移除。
- tokens.css 里 Lenis 相关类可清理。

## 验收标准
- 滚轮/触控板滚一下 = 整屏切换一屏,带过渡动画,切换中锁定不连跳。
- 方法页:翻入即播 4 卡入场,不随滚动进度变化。
- 精选作品:无自动轮播,仅 hover 展开。
- 超高段落(简历/目录)可内部滚动到底再翻屏,内容不被截断。
- `npm run build` 通过。

## 风险 / 待定
- 简历 Bento、目录网格可能超过 100vh:依赖内部滚动 + 边缘检测,需实测。
- 触控板惯性连发:靠 `locked` + 冷却时间,阈值需调。
- 移动端触摸阈值需实测。
- 用户后续替换内容后,各屏高度变化可能再需微调。
